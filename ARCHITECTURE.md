## ğŸ”„ Service Interactions

### Backend â†” Database
- **Connection Pool**: 20 max connections
- **Operations**: CRUD via Repository pattern
- **Transactions**: Used for atomic operations

### Backend â†” Redis
- **Session Storage**: JWT token blacklist (future)
- **Cache**: Frequently accessed data
- **Pub/Sub**: Real-time event distribution (future)

### Scraper â†’ Database
- **Direct Connection**: Separate pool
- **Batch Operations**: Bulk inserts for efficiency
- **Locking**: Prevents concurrent scraping

### Discord Bot â†’ Database
- **Read-Only**: Queries discord_config
- **Connection**: Simple pool (max 5)
- **Events**: Triggered by database changes

### Frontend â†” Backend
- **REST API**: All CRUD operations
- **WebSocket**: Real-time updates (Socket.IO)
- **Authentication**: JWT Bearer token
